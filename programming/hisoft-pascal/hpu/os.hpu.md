# OS.HPU

HiSoft Pascal може отримувати доступ до функцій операційної системи Enterprise за допомогою зовнішньої бібліотеки рутин. Для цього призначено файл **`OS.HPU`**. Щоб використовувати його, необхідно підключити його до нашої програми безпосередньо після розділу оголошення змінних за допомогою такої директиви:

```
{$F OS.HPU }
```

## Доступні процедури:

### TOGGLE(змінна)

Інвертує (перемикає) значення вказаної системної [змінної EXOS](../../system-info/info_exos-variables.md). Параметр має тип **INTEGER**.

### GETTIME(година, хвилина, секунда)

Записує поточний системний час у 24-годинному форматі в три змінні типу **INTEGER**.

### GETDATE(рік, місяць, день)

Записує поточну системну дату в три змінні типу **INTEGER**. Рік повертається у чотиризначному форматі.

### SETTIME(година, хвилина, секунда)

Встановлення системного часу. Усі змінні мають тип **INTEGER**. Години необхідно вказувати у 24-годинному форматі.

### SETDATE(рік, місяць, день)

Встановлення системної дати. Усі змінні мають тип **INTEGER**, рік необхідно вказувати чотирма цифрами.

### SYSINFO(b0, b1, b2, b3, b4, b5, b6, ver)

Запит стану системи. Записує у передані як параметри змінні типу **CHAR** параметри поточного стану системи:

 - **B0:** Номер shared сегмента,
 - **B1:** Кількість вільних сегментів,
 - **B2:** Кількість сегментів, виділених користувачеві,
 - **B3:** Кількість сегментів, виділених периферійним пристроям,
 - **B4:** Кількість сегментів, зарезервованих для системи,
 - **B5:** Загальна кількість (робочих) сегментів ОЗП (RAM),
 - **B6:** Кількість несправних (битих) сегментів ОЗП.
 - **VER:** Версія EXOS (у шістнадцятковому форматі, тобто для версії 2.1 поверне значення **33**).

Приклад програми системної інформації:

```
PROGRAM INFO;
VAR SEG0,SEG1,SEG2,SEG3,SEG4,SEG5,SEG6,VER:CHAR;
{$F OS.HPU }
BEGIN
	SYSINFO(SEG0,SEG1,SEG2,SEG3,SEG4,SEG5,SEG6,VER);
	WRITELN('Усього робочих сегментів RAM:  ',ORD(SEG5):3);
	WRITELN('Кількість вільних сегментів:   ',ORD(SEG1):3);
	WRITELN('Кількість розділюваних сегм.:  ',ORD(SEG0):3);
	WRITELN('Сегментів, виділених користув.:',ORD(SEG2):3);
	WRITELN('Сегментів, виділених периферії:',ORD(SEG3):3);
	WRITELN('Сегментів, виділених системі:  ',ORD(SEG4):3);
	WRITELN('Номер версії EXOS: ',ORD(VER))
END.
```

### CLEARFONT

Відновлює набір символів (шрифт) до початкового стану. Процедура не має параметрів.

## Доступні функції:

### BCD(NR)

Конвертує число у форматі **BCD** (двійково-десятковий код) типу **INTEGER** у звичайне десяткове число. Результат також повертається у форматі **INTEGER**.

### TOBCD(NR)

Конвертує десяткове число типу **INTEGER** у формат **BCD**. Результат також має тип **INTEGER**.

### ASK(N)

Повертає стан системної змінної з номером **N**. Параметр **N** та результат є значеннями типу **INTEGER**.

### GET(CH)

Зчитує один символ із відкритого каналу з номером **CH** і повертає його код як значення типу **INTEGER**. Зверніть увагу: **CH** є виразом типу **INTEGER**! Якщо в каналі немає символів, функція повертає `-1`. Функція **GET** працює також і на графічному екрані, але в цьому випадку вона повертає не код кольору, а індекс (номер) кольору, що належить графічній сторінці.

### JOY(NR)

Зчитує стан джойстика з номером **NR**. Повернене значення залежить від положення джойстика:

 - **1** — праворуч;
 - **2** — ліворуч;
 - **4** — вниз;
 - **8** — вгору;
 - **16** — натиснуто кнопку вогню (Fire).

Якщо джойстик нахилено по діагоналі або одночасно натиснуто кнопку вогню, функція повертає **суму** відповідних значень. Номер вбудованого джойстика — **0**, крім того, можна використовувати значення **1** (зовнішній порт EXT1) та **2** (зовнішній порт EXT2). Для вбудованого джойстика роль кнопки вогню виконує клавіша Пробіл. Вхідний параметр та повернене значення мають тип **INTEGER**.

### KEYPRESSED

Функція повертає **TRUE**, якщо в буфері клавіатури є натиснута клавіша. В іншому випадку вона повертає **FALSE**. Функція не має параметрів. Вміст буфера при цьому залишається незмінним, функція не вилучає звідти символ. Символ, згенерований натисканням клавіші, можна зчитати, наприклад, за допомогою функції **READKEY**.

### READKEY

Функція зчитує символ із клавіатури. Якщо буфер клавіатури порожній (немає натиснутих клавіш), функція чекає на натискання клавіші. Процедура не має параметрів, повертає значення типу **CHAR**.
