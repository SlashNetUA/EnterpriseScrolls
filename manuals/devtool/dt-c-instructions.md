(`*`)
Вказівники на процедури:

Повернене значення завжди int!
Синтаксис:
`int (*funcName)(... вхідні параметри ...) ;`

Друга дужка визначає вхідні параметри процедури.

Наприклад:
`int (*sprFunc)(); // вказівник на процедуру з іменем 'sprFunc' без вхідних параметрів.`
`int (*addFunc)(char a, int b) ; // вказівник на процедуру з іменем 'addFunc' з двома вхідними параметрами.`

Цілі числа розрізняються на знакові та беззнакові. Беззнакові цілі числа необхідно вказувати за допомогою модифікатора типу 'unsigned'.

Наприклад:
`char a;`  знаковий символ. Його значення знаходиться між -128 та 127.
`unsigned char a;` unsigned символ. Його значення знаходиться між 0 та 255.
`int a;` signed int. Його значення знаходиться між -32768 та 32767.
`unsigned int a;` unsigned int. Його значення знаходиться між 0 та 65535;

Вказівники завжди зберігаються в 16 бітах, представляють адресу елементів даних та інтерпретуються як беззнакові.

Класи сховища (крім extern), багатовимірні масиви
та складніші типи, такі як "`int *(*i)()`" (вказівник на функцію, що повертає цілочисельний вказівник), не реалізовані. Хоча багатовимірні масиви не дозволені, масиви з елементами масиву, що складаються з вказівників на масиви, можуть досягти подібного ефекту.

Структури можна оголошувати за допомогою такого синтаксису:

```
struct st
{
  int x ;
  char c[12] ;
  struct other s ;
  struct st *st_pointer;
} x, *y, z[5] ;
```

Тег структури є обов'язковим ('st'). Оголошення структур можуть бути вкладеними, хоча члени вкладених структур мають однакову назву. Структури не можуть містити екземпляри самих себе, але вони можуть містити вказівники на екземпляри самих себе.

Структури необхідно оголошувати до першого посилання на них. Передача та присвоєння структур не підтримуються. Це означає, що єдиним допустимим входженням структури у список аргументів функції (процедури) є вказівник.

Мітки структур, члени структури та звичайні змінні є окремими просторами імен. Об'єднання розглядається як структура, тому застосовуються ті самі правила та обмеження.

## ЕЛЕМЕНТАРНІ КОМПОНЕНТИ МОВИ

**масиви** (наприклад: array[expression])
**функції** (наприклад: function(arg1,arg2,...,argn))
**структурні елементи** (наприклад: structure.member)
**елементи-вказівники на структуру** (наприклад: st_pointer->member)
**константи**
 - **десяткове ціле число** (наприклад: 1456, -8192)
 - **шістнадцяткове ціле число** (наприклад: 0xfe, 0X12CD, 0x04A)
 - **десяткове значення з плаваючою комою** (наприклад: 1.0, 2., .3, 340.2e-8)
 - **рядок** (наприклад: "приклад тексту")
 - **один символ** (наприклад: 'a' або, скажімо, 'Z')
метод **sizeof()**
**локальні змінні**
**глобальні змінні**
**Блок коментарів** між знаками `/* ... */`, навіть через кілька рядків.
**Рядок коментарів** `//` з кінця рядка.

Оголосіть усі змінні перед використанням. Змінні, оголошені поза функціями (процедурами), є глобальними. Це означає, що вони видимі та можуть використовуватися всередині будь-якої процедури.
Локальні змінні мають бути оголошені на початку складеного оператора.
(за винятком складеного оператора switch.) Складені оператори мають бути укладені у фігурні дужки '{}'. Область дії локальних змінних обмежена складеним оператором або функцією, в якій вони оголошені. Ключове слово extern може використовуватися для оголошення зовнішніх змінних та функцій. Значущими є лише перші 48 символів імен змінних та функцій.

Наступні об'єкти GLOBAL можуть бути ініціалізовані при першому оголошенні:

символи, цілі числа (16-бітні цілі числа), вказівники на символи, символьні та цілочисельні масиви, масиви, що містять вказівники на символи, структури.

Деякі приклади ініціалізації:

```
char c = 'a' ;
int i = 0 ;
char *message = "Привіт, світе\n" ;
char bye[] = "До побачення" ;
int x[7] = { 0, 1, 2, 3, 4, 5, 6 } ;
char *mon[] = { "січ", "лют", "мар", 0 } ;

struct st { char c; int x; char *cp } ;
struct st st1 = { 'q', 23, "рядок"} ;
struct st *st2 = { 'r', 42, "інший рядок"} ;
struct st st3[] = {
{'a', 1, "a"},
{'b', 2, "b"} } ;
```

Локальні змінні (наприклад, у процедурах) не можна ініціалізувати під час оголошення!

Розміри масивів, що ініціалізуються, можна вказати в оголошенні. У цьому випадку неініціалізовані записи будуть встановлені на нуль.

Або ж розмір масиву можна залишити порожнім, і в цьому випадку розмір масиву визначається кількістю елементів масиву під час ініціалізації.

Неініціалізовані глобальні змінні мають бути встановлені на нуль. Локальні змінні не можна ініціалізувати, а їхні початкові значення та вміст невизначені.

Вказівники символів можна ініціалізувати лише рядковою константою або нулем. Єдиними членами структури, які можна ініціалізувати, є символи, цілі числа та вказівники на char.

Наступні символи з так званих послідовностей ESC можна використовувати в лапках або апострофах: "`\b`", "`\t`", "`\r`", "`\f`" та "`\n`", які дорівнюють **8**, **9**, **10**, **12** та **13** відповідно. Вони представляють 13-символьні коди.  
Зверніть увагу, що "`\n`" означає повернення каретки, а не більш поширений перехід рядка. Для нового рядка можна використовувати послідовність "`\r\n`".  
Будь-який символ може бути представлений тризначним вісімковим символом: "`\ooo`". Усі інші символи після зворотної склесної риски втратили своє спеціальне значення, тому лапки також можна використовувати в лапках. Отже: '`\"`'.

Значення рядка в лапках – це вказівник на рядок, що завершується нульовим (0) байтом, десь у пам'яті.
Оператор `sizeof()` повертає розмір об'єкта в байтах.
У ньому можна використовувати лише слова "int", "char", "double" або назви структур у форматі "struct st", де st – назва структури. Назви змінних використовувати не можна.

### Умовний оператор:

`condition ? 'if true' : 'if false'`

Перетворення між числами з плаваючою комою та цілим числом відбувається автоматично для виразу, що повертається функцією.
Перетворення цілочисельних змінних у змінні з плаваючою комою відбувається автоматично в аргументі будь-якої функції з плаваючою комою.
В іншому випадку можна використовувати процедури "float(jj)" та "ifix(yy)" (як у FORTRAN). Цілочисельні константи не можна використовувати у виразах з плаваючою комою. Компілятор попереджає про це.
Аргументи операторів, що містять лише цілі числа, перевіряються програмою на цілочисельність. Перевірка типу фактичних параметрів, що передаються викликам функцій, не виконується.

Коли ціле число додається до вказівника, приріст масштабується на розмір об'єкта, на який вказує. Таким чином, додавання n до вказівника вказує на n-й об'єкт у пам'яті, незалежно від розміру об'єкта. Коли ми віднімаємо два вказівники (вони можуть бути лише вказівниками одного типу!!), різниця зменшується так, що результат дорівнює кількості елементів між ними. Результат має сенс лише тоді, коли вказівники вказують на один і той самий масив: у цьому випадку ми отримуємо такий результат:
кількість елементів між вказівниками.

Оператор крапка з комою можна використовувати для розділення виразів у списку виразів. Вирази у списку обчислюються справа наліво.

### Керувальні оператори

Існує три типи:
 - гілки (if та switch)
 - оператори циклу (while, for, do/while)
 - goto

### Оператор if

Форма:

`оператор if(умова); оператор else2;`

На відміну від інших мов програмування, C не має ключового слова then (саме тому умова має бути взята в дужки). Оператор else можна пропустити.

Після if та else може бути оператор, який також може бути блоком (і майже завжди є). Якщо оператор також є if, і два if мають else, він належить до внутрішнього. Бажано уникати таких ситуацій, використовуючи { }

.

У C немає логічних змінних або виразів (реляційні операції також повертають цілочисельні значення), тому умова є цілочисельним арифметичним виразом, якщо її значення дорівнює 0, умова не виконується (інакше розгалуження), але якщо її значення дорівнює 0, вона виконується.

Приклад:
```
if(a > 5)
{
  x = 3;
  y = 2;
}
else
{
  x = 8;
  y = 9;
}
```
також можна записати так (хоча це не рекомендується через легку можливість плутанини):

`if(a > 5) x = 3, y = 2; else x = 8, y = 9;`

### Оператор switch

Багатонаправлене розгалуження відповідно до можливих значень цілочисельного арифметичного виразу. Можливі значення є цілочисельними константами.

Формат:

```
switch(вираз)
{ case value1: ...
    break;
  case value2: ...
    break;
  default: ...
}
```

Цілочислові константи після case повинні бути різними. Керування передається до case, значення якого збігається зі значенням виразу. Якщо жоден з них не збігається, керування передається до гілки за замовчуванням. Default можна пропустити, і в цьому випадку switch нічого не робить, якщо значення виразу відрізняється від кожного case.

Break завершує оператор switch. Необов'язково розміщувати його в кінці кожної гілки: у цьому випадку гілка «переходить» до наступної. Оскільки це зазвичай помилка програми, якщо вона навмисна, доцільно вказати це в коментарі.

### Оператор while

Формат:

`оператор while(умова);`

Умова, як і оператор if, є цілим виразом, а оператор – тіло циклу – майже завжди є блоком.


Спочатку обчислюється умова. Якщо значення не дорівнює 0, оператор виконується, а потім умова обчислюється знову, доки умова не набуде значення 0.

Оператор break виходить з циклу. Оператор continue виходить з тіла циклу та продовжує виконувати умову.

Нескінченний цикл:

`while(1)`
Крапка з комою позначає порожній оператор.

### Оператор for
Форма:

`for(передвираз; умова; післявираз) оператор;`

Його ефект такий самий, як якби ми написали це:

```
передвираз;
while(умова)
{
  оператор;
  поствираз;
}
```

Якщо умову пропущено, вона вважається істинною. Будь-який з двох виразів можна пропустити; for(;умова;) ідентичний оператору while(умова). Оператор також можна пропустити, але крапка з комою не є (порожній оператор). Це досить поширене явище: завдання циклу виконується заголовком циклу.

Оператор break виходить з циклу. Оператор continue виходить з тіла циклу та продовжує виконання пост-виразу.

Приклад: ми підраховуємо, скільки значень 1 є в tomb, і поміщаємо їх у змінну samll.

```
int tomb[1000],samll,*mut;
for(mut=tomb,samll=0; mut-tomb < sizeof(tomb)/sizeof(int); mut++) {
  if(*mut == 1)
  samll++;
}
```
sizeof(tomb) – це розмір tomb у байтах. sizeof(int) – це довжина цілочисельного типу в байтах. Частка двох дає кількість елементів у tomb, тому його потрібно було записати лише один раз. mut-tomb дає кількість елементів масиву, на яких знаходиться цикл.

Після завершення циклу значення змінної mut буде tomb+1000. У C немає справжніх змінних циклу; значення змінних в операторі for можна використовувати без зв'язування навіть після завершення циклу або після виходу з break, на відміну від більшості мов.

Нескінченний цикл:

`for(;;)`

### Оператор do/while
Третя форма оператора циклу. Вона відрізняється від while тим, що спочатку виконується тіло циклу, а потім оцінюється умова. Вона рідко використовується в C.

Формат:

`оператор do; умова while;`

### Оператор goto

Структурне програмування було винайдено, щоб уникнути оператора goto та зробити програму більш читабельною, тому використання goto взагалі не рекомендується.

Формат:

`мітка goto;` Мітка - це ім'я, за яким йде двокрапка. Goto змушує програму продовжувати роботу з оператором, позначеним міткою.
Приклад:

`мітка goto; ...`

## Функції

Функції обчислюють значення на основі параметрів, отриманих від програми, що викликає, та повертають його програмі, що викликає. Оголошення функції забезпечує правильну інтерпретацію параметрів та повернених значень.

У мові C немає процедур, але програма, що викликає – через правила арифметичних інструкцій – не зобов'язана використовувати повернене значення функції, тому функції можна використовувати як процедури. На практиці використання функцій як процедур викликало плутанину, тому в ANSI C було введено тип void, який не можна використовувати для жодної операції. Це означає, що функцію, яка повертає тип void, практично можна вважати процедурою.

### Оголошення функції
Згідно з базовим синтаксисом C, необхідно вказати тип поверненого значення, назву функції, а також кількість і тип параметрів. Останні взяті в дужки. Також можна написати функцію без вхідних параметрів, і в цьому випадку між дужками нічого не вводиться. (Для цього потрібні дужки!) Компілятор Small-C/Plus не вимагає вказівки типу повернення, його можна пропустити. Під час повернення вміст регістру 'HL' буде поверненим значенням.

Наприклад:
`int strlen(char *str);`

### Визначення функції

Він визначає інструкції функції. Він складається з двох частин: заголовок визначає параметри та тип значення, а блок, що йде за ним, визначає інструкції. Заголовок може бути записаний у двох формах, подібних до двох типів визначень функцій. Оригінальна форма, подібна до визначення типу:

`int memcmp(string1,string2) const char *string1,*string2; {...}`

Інша форма:

`int strcmp(const char *string1, const char *string2) {...}`

Примітка:
1) Сучасні компілятори C майже приймають лише другу форму. Small-C/Plus був підготовлений для оригінальної, тобто першої, форми визначення. Оскільки я сам віддаю перевагу використанню другої (більш поширеної) форми, я створив простий метод перетворення всередині компілятора C, який працює як секрет під час перетворення та створює першу форму, необхідну компілятору, з формули, заданої вдруге.

2) У цьому варіанті C – оскільки він не знає модифікатора 'void' – тип повернутого значення можна пропустити.

Функція визначає своє повернене значення за допомогою виразу після оператора return. Вираз автоматично перетворюється на тип функції. У випадку функції типу void вираз після оператора return пропускається, і оператор return не потрібно писати, якщо це останній оператор функції перед закриваючим }.

Якщо функція повертає вказівник, вона не може вказувати на динамічні дані (простір стеку). Під час повернення з функції простір пам'яті динамічних змінних звільняється, і простір може змінитися будь-коли.

Змінні, створені локально у визначенні функції, розміщуються на STACK. Їхнє значення втрачається після виклику функції. На початку кожної функції необхідно оголосити використані локальні змінні. На відміну від глобальних змінних, оголошення не можна поєднувати з визначенням!!

Наприклад:

```
int Multiply(int a, int b)
{
int local1, local2; // Оголошення має бути попереду!
int hibas = 12; // Це неправильне - поєднане з оголошенням - визначення!!!!! Не використовується!!!

local1 = a; // Визначення може бути лише після оголошення.
local2 = b;

....
}
```

### Виклик функції
Параметри повинні бути записані в дужках після назви функції, розділені комою. Дужки повинні бути записані, навіть якщо функція не має параметрів, тобто дужки є операцією виклику функції.
Параметри передаються за значенням, тобто функція отримує копію значень. Це також означає, що вона не може змінити значення змінних, що викликають. Це долається передачею адреси (вказівника) змінної функції.

Приклад викликів функцій:

```
a = Ossead(12,c+4);
cls();
```

### Вказівник функції
Вказівник функції — це змінна, яка зберігає адресу пам'яті функції, на якій може бути виконана операція виклику функції (()).

Вказівник функції також можна оголосити двома способами:

`int (*comp)();`  
`comp = strcmp;`

Тоді, наприклад, вирази comp("one",valt) та strcmp("one",valt) матимуть абсолютно однаковий ефект.

## Оператори препроцесора
Оператори препроцесора змушують перший прохід компіляції вносити зміни до вихідної програми, в результаті чого програма на C не містить операторів препроцесора.
Оператори препроцесора не є довільною формою: вони починаються з # на початку рядка та закінчуються кінцем рядка. Рядок продовження можна записати з \ в кінці рядка.

### \#include

Форма:

`#include <ім'я_файлу>`  
`#include "ім'я_файлу"`  

Препроцесор замінює оператор include вмістом файлу із зазначеним ім'ям (заголовковий файл) (ніби його було введено в програму). Різниця між двома формами полягає в тому, що перша шукає зазначений файл серед файлів компілятора (у Linux, наприклад, у каталозі /usr/include), тоді як друга шукає зазначений файл у тому ж каталозі, що й програма на C (її власний заголовковий файл).
Ім'я файлу може містити шлях, а його розширення - .h згідно з усталеними звичаями. Також прийнято, що заголовковий файл містить лише оголошення даних, функцій та типів (окрім інструкцій препроцесора), а не виконувані інструкції.
Використання заголовкових файлів дозволяє записувати оголошення в одному місці. Програми з вихідним кодом повинні посилатися на неї лише за допомогою оператора \#include.
### \#define

Присвоює значення змінній препроцесора. Препроцесор підставляє значення текстово замість змінної. Наприклад, якщо на розмір масиву посилаються в кількох місцях програми:

```
#define TOMB SIZE 100  

int tomb[TOMB SIZE],i;
for(i=0; i < TOMB SIZE; i++) ...
```

Це дозволяє змінювати розмір масиву в одному місці програми.
Значення \#define може бути будь-яким текстом. Якщо ви хочете назвати вираз C таким чином, наполегливо рекомендується взяти вираз у дужки. Під час використання виконується текстове копіювання, перевірка пріоритету не виконується.
Оператор \#ifndef можна використовувати для перевірки існування змінної (а також для присвоєння їй значення за замовчуванням, якщо під час компіляції значення не було задано).

### \#ifdef, \#ifndef

Перевіряє існування змінної препроцесора. Його формат:

`#ifdef preproc variable ... #else ... #endif`

Гілку \#else можна пропустити. Кілька операторів можна вкладати один в одного. \#ifdef виконується, якщо змінна існує, \#ifndef — якщо ні.

За допомогою цієї команди можна пропустити вихідний код програми. Гілку \#else можна пропустити. Якщо її пропустити, текст (програма), записаний до \#endif, буде включено до програми лише за умови виконання умови.

### \#asm - \#endasm

Рядки між цими директивами будуть включені до згенерованого asm-коду без змін.

### \#options - \#endoptions

Цей розділ використовується для передачі параметрів компіляції до згенерованого asm-коду, який зазвичай складається з директив \#define. Однак, ці визначення ВІДНОСЯТЬСЯ ЛИШЕ ДО КОМПІЛЯТОРА АСЕМБЛЕРА, вони не впливатимуть на код C.

Пару директив \#options - \#endoptions можна розмістити лише на початку нашої програми C!

### \#pragma

Багатофункціональна директива. Її основна функція — керування компонувальником, але вона також може керувати роботою різних експортерів DevTool — наприклад, включенням графічного робочого аркуша.

#### \#pragma mblock <адреса> - \#pragma endmb

Компонувальник розміщує глобальні змінні, зазначені між цією парою директив, у пам'яті, починаючи з вказаної адреси. Типовим випадком використання є переміщення робочих змінних та робочих масивів нашої програми до невикористаної області пам'яті компонувальника. Наприклад, якщо наша програма не використовує відображення символів операційної системи, ми можемо перемістити робочі змінні до робочої області, починаючи з 0x740, де компонувальник зберігає перевизначені символи BASIC.

Ми можемо реалізувати це так:

```
// Цей масив або змінні будуть розміщені на місці перевизначених символів BASIC.

#pragma mblock 0x740
char myArray[100];
int prnPositionX, prnPositionY;
#pragma endmb
```

#### \#pragma link [forward / front / backward / back]

Через керування пам'яттю TVC нам може знадобитися якимось чином вплинути на те, в яку частину пам'яті будуть перенаправлені деякі частини нашої програми. Типовий приклад: розташування частин програми, які обробляють відеопам'ять, може бути критичним, оскільки якщо програмний код, який її обробляє, розміщується на другій сторінці пам'яті під час компіляції, то коли ми сторінкуємо відеопам'ять за допомогою сторінкового підкачування пам'яті, наш програмний код стає недоступним для z80 - повністю або частково. Це робить речі особливо заплутаними, якщо ми також використовуємо бібліотеки, які зв'язані з автокомпоновувачем. Під час налаштування компілятора C для TVCDeveloperTool можна вказати, що певні частини бібліотеки повинні бути включені в нашу програму послідовно, тобто відповідно до позиції, зайнятої в програмі C (де знаходиться компілятор під час компіляції) – це стан за замовчуванням, або процедури бібліотеки автокомпонування завжди повинні розміщуватися в кінці нашої програми.

Однак, все ще можливо якимось чином вплинути на позицію заданої частини програми.
Для цього призначена директива \#pragma link.

`#pragma link forward`  
`#pragma link front` 

Під час компіляції лінкер спробує розмістити процедуру, що йде після цієї директиви, якомога далі назад у нашій програмі.

`#pragma link backward`  
`#pragma link back`  

Під час компіляції лінкер спробує розмістити процедуру, що йде після цієї директиви, якомога далі назад у нашій програмі.

Ці два параметри впливають на процедуру, що йде безпосередньо після директиви. Якщо ми розмістимо її перед глобальним блоком даних, порожня директива \#pragma link в кінці блоку даних відновить базову операцію лінкера. (У випадку процедури її можна пропустити)

`#pragma rem <рядок_контролю_експортера>`

Ця директива не впливає на процес компіляції або лінкування. Її використання призначене для допомоги в експорті графічного робочого аркуша або інших експортерів у код TVC з DevTool версії 3.2.0.

Наприклад:

```
#pragma rem START_GFX_DATA
..
..
експортер розмістить тут дані для графіки, елементів треку, анімації
..
..
#pragma rem END_GFX_DATA
```
